## ğŸ¯ Common Actions

### Publishing Changes
1. Edit content in any field
2. Click **Save** (saves draft, not published)
3. Click **Publish** (commits to GitHub, triggers rebuild)
4. Wait 2-3 minutes for site to rebuild
5. Check your live site for changes

### Adding New Items
1. Go to collection (Projects, Services, Testimonials)
2. Click **New [Item]** button (top right)
3. Fill in all required fields (marked with *)
4. Click **Publish**

### Deleting Items
1. Open the item you want to delete
2. Click **Delete** button (top right)
3. Confirm deletion
4. Item is removed and changes are published

### Uploading Images
1. Click on image field
2. Click **Choose an image**
3. Either:
   - Click to browse your computer
   - Drag and drop image onto the area
4. Image uploads to `public/img/`
5. Image path is automatically set

### Reordering Items
- **Services & Testimonials**: Change the "Order" number
  - Lower numbers appear first (1, 2, 3...)
- **Skills Categories**: Use â–² â–¼ arrows to move up/down
- **List Items**: Use â–² â–¼ arrows or drag to reorder

## ğŸ’¡ Tips for Using the Interface

### Required Fields
- Fields marked with * are required
- You can't publish without filling them
- Red error messages show if missing

### Rich Text Editor (Markdown)
- **Bold**: Select text, click [B]
- **Italic**: Select text, click [I]
- **Link**: Select text, click [Link], enter URL
- **Headings**: Click [H1] or [H2]
- **Lists**: Click [â€¢] for bullets, [1.] for numbers

### Image Best Practices
- Optimize before uploading (< 500KB)
- Use descriptive filenames
- Supported formats: JPG, PNG, GIF, WebP, SVG
- Recommended size: 1200x800px for projects

### Saving vs Publishing
- **Save**: Saves draft, not live on site
- **Publish**: Commits to GitHub, triggers rebuild
- Always click **Publish** to make changes live

### Preview Changes
- CMS doesn't have live preview
- After publishing, wait 2-3 minutes
- Visit your live site to see changes
- Or test locally: `npm run dev`
